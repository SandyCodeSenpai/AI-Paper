# Llama 2: Open Foundation and Fine-Tuned Chat Models

## TL;DR

Paired release of base models (7B / 13B / 34B / 70B, trained on 2T tokens) and chat models (Llama 2-Chat at the same sizes). Architecture is mostly LLaMA 1 — pre-norm + RMSNorm + SwiGLU + RoPE — with two changes: **4K context (was 2K)** and **[concepts/gqa](../concepts/gqa.md)** on the 34B and 70B. The actual contribution is ~30 pages documenting the post-training pipeline: SFT, two-RM design, iterative RLHF (Rejection Sampling V1–V4 then PPO V5), Ghost Attention, and a three-step safety pipeline. Llama 2-Chat-70B is competitive with ChatGPT (gpt-3.5-turbo-0301) on human-rated helpfulness, and ships under a **commercial license** — ending LLaMA 1's research-only restriction.

## Problem & Motivation

- **Open the alignment recipe.** Pre-Llama-2, the post-training details for ChatGPT / Claude were sketchy in public. Open base models existed (LLaMA 1, BLOOM, Falcon) but no equivalent of GPT-3.5-turbo. Llama 2 closes that.
- **Inference economics at long context.** MHA's KV cache becomes a bottleneck at 4K context and large batch — GQA addresses this without quality loss.
- **Commercial licensing.** Without it, LLaMA 1 couldn't seed startups or production usage. Llama 2 enables the late-2023 ecosystem.

## Background / Prior Work

- [sources/llama1-2023](llama1-2023.md) — Llama 2 inherits the architecture wholesale
- InstructGPT (Ouyang 2022) — established SFT + RM + PPO recipe
- Anthropic HH-RLHF (Bai 2022) — preference data, helpfulness/harmlessness split
- Chinchilla (Hoffmann 2022) — scaling laws referenced for token count decisions
- MQA (Shazeer 2019), GQA (Ainslie 2023) — KV-cache compression precedents
- Context Distillation (Askell 2021) — borrowed for safety preprompt distillation

## Key Contribution

1. **GQA enters the open ecosystem.** First major open release with GQA at scale; ablation in A.2.1 makes it the reference point everyone cites.
2. **First detailed open RLHF playbook.** Two-RM design, iterative collection, Rejection Sampling vs PPO, KL-penalty piecewise reward, Ghost Attention, safety context distillation — all documented at reproducible detail.
3. **Commercial license** — practical unlock for the open ecosystem.

## Method

### Architecture (§2.2 + Appendix A.2.1)

Same as LLaMA 1 plus:

- **Context length:** 2048 → 4096 (Table 16/17 ablation: improves long-context tasks, no regression on short).
- **[concepts/gqa](../concepts/gqa.md)** on 34B and 70B: 64 Q heads × 8 KV heads (group size 8). 7B and 13B keep MHA. FFN dim bumped ~1.3× to compensate for parameter loss in attention.

| | 7B | 13B | 34B | 70B |
|---|---|---|---|---|
| dim | 4096 | 5120 | 8192 | 8192 |
| layers | 32 | 40 | 48 | 80 |
| heads (Q) | 32 | 40 | 64 | 64 |
| KV heads | 32 (MHA) | 40 (MHA) | 8 (GQA) | 8 (GQA) |
| context | 4096 | 4096 | 4096 | 4096 |
| tokens trained | 2.0T | 2.0T | 2.0T | 2.0T |
| LR | 3e-4 | 3e-4 | 1.5e-4 | 1.5e-4 |

### Pretraining data

"New mix of publicly available online data," up-sampling factual sources to "increase knowledge and dampen hallucinations." 89.7% English, 8.4% unknown (mostly code), <0.2% per other language. **No Meta user data.** Same SentencePiece BPE tokenizer as LLaMA 1, 32K vocab, digits split, byte fallback. Total: 2T tokens, 4M batch.

### Post-training pipeline (§3)

```
Pretraining → Llama 2 base
    ↓ SFT on 27,540 hand-curated examples
Llama 2-Chat (initial)
    ↓ Iterate V1...V5:
       collect preference data → train RMs → rejection-sample-tune (V1-V4) → +PPO (V5)
Llama 2-Chat (final)
```

#### [Supervised Fine-Tuning](../concepts/sft.md)

**Quality > quantity.** They abandoned millions of third-party instruction examples in favor of 27,540 vendor-written prompt+response pairs. Cosine LR 2e-5, weight decay 0.1, batch 64, seq 4096, 2 epochs, **loss masked on user tokens**.

#### Human preference data

- **Two streams:** helpfulness, safety. Safety gets a tri-class meta-label (chosen-safe-other-unsafe / both-safe / both-unsafe — never reverse).
- 4-point preference scale: *significantly / better / slightly / negligibly* better.
- Weekly batches; each batch uses the *latest* checkpoint's outputs (keeps RM in distribution).
- 1.4M Meta comparisons + 1.5M open-source = **2.9M total**.

#### [Reward Modeling](../concepts/reward-modeling.md) — two separate RMs

Helpfulness and safety conflict; one RM can't handle both. Trained from chat checkpoint (LM head → scalar head — RM "knows what the model knows," prevents reward hallucinations).

**Margin-augmented binary ranking loss:**

$$L = -\log \sigma(r_\theta(x, y_c) - r_\theta(x, y_r) - m(r))$$

where m(r) scales with preference strength. Improves accuracy on clearly-separable pairs (Table 28).

Final RM accuracies (Table 7): Helpfulness RM 63.2 avg, Safety RM 64.3 avg — both beating GPT-4-as-judge (63.0 avg).

#### [Rejection Sampling](../concepts/rejection-sampling-finetuning.md) (V1–V4)

For each prompt, sample K candidates from current model, RM-score them, keep the top one, fine-tune model on these like SFT. K and temperature are tuned (Figure 7-8): with V5 RLHF, optimal sampling temperature is **T=1.2–1.3**, higher than SFT's T=1.0.

**Distillation through RS:** small models are trained on rejection-sampled outputs from the largest (70B), which distills 70B-quality answers into them.

#### [PPO](../concepts/ppo.md) (V5 only)

Piecewise reward (safety overrides if prompt is flagged or safety score < 0.15):

$$R_c(g | p) = \begin{cases} R_s(g | p) & \text{if IS\_SAFETY}(p) \text{ or } R_s(g|p) < 0.15 \\ R_h(g | p) & \text{otherwise} \end{cases}$$
$$R(g | p) = \tilde{R}_c(g | p) - \beta \, D_{KL}(\pi_\theta(g|p) \,\|\, \pi_0(g|p))$$

with logit-then-whiten transform (`R̃ = WHITEN(LOGIT(σ(R_c)))`) for stability with KL penalty. KL coefficient β = 0.01 (7B/13B), 0.005 (34B/70B). PPO clip 0.2, batch 512, mini-batch 64, 1 grad step per mini-batch, constant LR 1e-6, AdamW with same betas as pretraining.

200–400 PPO iterations per RLHF version. ~330 sec/iteration on 70B. FSDP with weight-consolidation trick for fast generation.

#### [Ghost Attention (GAtt)](../concepts/ghost-attention.md)

Multi-turn consistency hack. During fine-tuning, synthetically prepend the system instruction to every user turn in a multi-turn dialogue. Loss is zeroed on intermediate assistant turns. The model learns to attend to the system message throughout the dialogue. No architectural change; just data augmentation. Figure 10 shows post-GAtt attention to system message stays large across all turns.

### Safety (§4)

Three-step pipeline mirrors helpfulness:

1. **Supervised safety fine-tuning** — adversarial prompts + safe demonstrations folded into general SFT.
2. **Safety RLHF** — Safety RM, piecewise-combined with helpfulness RM in PPO.
3. **Safety context distillation** — generate response with "You are a safe and responsible assistant" preprompt, then fine-tune to produce same response without preprompt. **Only kept on samples where Safety RM scores higher than original** — distillation can degrade good responses.

Red-team metric γ (adversarial prompts per person-hour to elicit a violation): 1.8 → 0.45 across multiple rounds, 350+ red-teamers.

## Experiments

### Base-model academic benchmarks (Tables 3, 4, 19–25)

Llama 2 base improves Llama 1 across the board:

| | Llama 1-65B | Llama 2-70B |
|---|---|---|
| MMLU 5-shot | 63.4 | **68.9** |
| BBH 3-shot | 43.5 | **51.2** |
| GSM8K 8-shot | 50.9 | **56.8** |
| HumanEval pass@1 | 23.7 | **29.9** |
| TriviaQA 1-shot | 84.5 | **85.0** |

Vs closed: close to GPT-3.5 on MMLU (68.9 vs 70.0) and GSM8K (56.8 vs 57.1); large gap on HumanEval (29.9 vs 48.1); well below GPT-4 (86.4 / 92.0 / 67.0).

### Chat-model human evaluation

- Llama 2-Chat-70B vs ChatGPT-0301 (4K prompts, 3 raters each): 36% win, 31.5% tie, 32.5% loss
- Llama 2-Chat-70B vs PaLM-Bison: 53.0% win, 24.6% tie, 22.4% loss
- Llama 2-Chat-7B vs MPT-7B-chat: 61.1% win, 20.9% tie, 18.0% loss
- Llama 2-Chat-13B vs Vicuna-13B-v1.1: 45.4% win, 29.8% tie, 24.9% loss

### Safety evaluation

- Llama 2-Chat-70B: ~4% violation rate (Figure 17); ChatGPT ~7%, PaLM-Bison ~28%, Vicuna-33B ~38%
- Llama 2-Chat-7B/13B: **0.00% ToxiGen toxicity** (lowest of all)
- TruthfulQA truthful·informative: 64.14 (70B), 67.20 (34B)

### Reward-model scaling (Figure 6)

Both data and model size matter; **the RMs have not yet saturated** at 14 batches of preference data. Helpfulness RM accuracy continues to climb; 70B RM clearly outperforms 13B RM.

## Results

The headline behavioral results are above. The deeper findings:

- **Iteration matters more than algorithm.** RLHF V1–V5 each shows clear progression. Final win-rate vs ChatGPT improves from V2 (~30%) to V5+PPO (>60% per Meta judge, >36% per GPT-4 judge).
- **Safety doesn't tank helpfulness.** Adding safety RLHF doesn't move helpfulness RM scores meaningfully (Figure 14, right side).
- **In-context temperature rescaling:** RLHF teaches the model to use temperature *differently* on factual vs creative prompts (Figure 21, §5.1).
- **Tool emergence:** chat models call APIs without being trained to (mentioned, not formally evaluated).

## Limitations

Authors acknowledge:
- Knowledge cutoff Sept 2022 (most pretraining), July 2023 (some fine-tuning data)
- 89.7% English; non-English performance is materially worse
- Hallucinations remain
- Safety evaluations biased toward English and toward Llama 2's own content standards
- 34B not released — "insufficient time to red team"

Not stated but worth noting:
- No DPO (DPO postdates Llama 2)
- No long-context strategy beyond 4K
- No native multimodality
- False refusals on borderline prompts rise with safety data fraction (Section 4.2.3)

## My Takeaways

- **The architecture is a footnote; the pipeline is the paper.** The model architecture changed only slightly from LLaMA 1; the alignment recipe is what made this paper foundational.
- **Two-RM > one-RM** for helpfulness/safety is a robust finding — directly informs DeepSeek-R1's later reward-shaping decisions.
- **Rejection sampling fine-tuning is underrated.** Got Llama 2-Chat through V1–V4 without PPO; PPO only added at the end. Many subsequent open-model teams have followed this "RS first, PPO last" pattern.
- **Quality > quantity for SFT** is now received wisdom (Phi, Zephyr, etc.) — Llama 2 documented it convincingly with the 27.5K-example final SFT set.
- **GAtt is a clever hack** — pure data augmentation, no architecture change, fixes a real multi-turn problem. Worth remembering as a pattern.

## Open Questions

- How much of Llama 2-Chat's quality comes from preference *data quality* vs the algorithmic choices? Hard to disentangle without a counterfactual.
- The 34B was trained but not released — what would the cost/value tradeoff look like for a 30B-class GQA model? (DBRX, Qwen-32B later answered this differently.)
- Does iterative RLHF fundamentally beat single-pass RLHF, or is it just a way to keep the RM in distribution? Not directly ablated.
- Is the false-refusal cost of safety RLHF avoidable, or fundamental to the RM-based approach?

## Glossary

- **GQA (Grouped-Query Attention):** Attention variant where K and V projections are shared across groups of query heads. Reduces KV cache without the quality regressions of MQA. See [concepts/gqa](../concepts/gqa.md).
- **MHA / MQA / GQA:** Multi-head / Multi-query / Grouped-query attention. Trade off KV cache size against expressiveness.
- **KV cache:** At inference time, K and V vectors for prior tokens are cached so each new token's attention costs O(seq_len), not O(seq_len²).
- **SFT (Supervised Fine-Tuning):** Train base model on curated prompt+response pairs with autoregressive loss masked on the user prompt.
- **RLHF (Reinforcement Learning from Human Feedback):** Multi-stage pipeline: collect human preferences → train reward model → optimize policy. See [concepts/rlhf](../concepts/rlhf.md).
- **Reward Model (RM):** Scalar regression head fine-tuned from a chat checkpoint, predicting human preference. See [concepts/reward-modeling](../concepts/reward-modeling.md).
- **PPO (Proximal Policy Optimization):** Policy-gradient RL algorithm with KL penalty and clipping. See [concepts/ppo](../concepts/ppo.md).
- **Rejection Sampling fine-tuning:** Sample K candidates, take the highest-RM one, fine-tune the model on those. See [concepts/rejection-sampling-finetuning](../concepts/rejection-sampling-finetuning.md).
- **Ghost Attention (GAtt):** Synthetic data trick — prepend system instruction to every user turn during multi-turn fine-tuning. See [concepts/ghost-attention](../concepts/ghost-attention.md).
- **Context Distillation:** Generate output with a guiding preprompt, then fine-tune model to produce same output without the preprompt.
- **FSDP (Fully Sharded Data Parallel):** PyTorch's sharded data-parallel implementation; shards parameters, gradients, and optimizer states across data-parallel workers.
